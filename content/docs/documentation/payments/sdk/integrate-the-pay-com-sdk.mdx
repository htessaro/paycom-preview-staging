To integrate the SDK into your app, start by setting up the shared configuration required to use the universal and headless integration modes. By the end of this guide, you will have a `pay` instance ready to use in your app to [create a payment charge](/docs/payments/guides/create-a-charge).

<Callout type="info">
  The base URLs used in this guide are for the sandbox environment. For production, replace `https://api.staging.pay.com` with `https://api.pay.com`. see the [API reference](/docs/api-reference) for a full list of environment URLs,
</Callout>

Before you begin, generate a public API key from your [dashboard](https://pay.com/dashboard). See the [API keys guide](/docs/payments/03-get-started) for detailed instructions.
<Steps>
  <Step>
    ## Install the SDK
    Add the SDK to your project. You can install it using a package manager (npm or yarn) or load it directly in the browser using our CDN.

    <Tabs items={['npm / yarn', 'CDN']}>
      <Tab value="npm / yarn">
        Install the SDK using npm or yarn. The package includes TypeScript definitions.

        ```bash
        yarn add @pay-com/js
        # or
        npm install --save @pay-com/js
        ```
      </Tab>
      <Tab value="CDN">
        Include the Pay.com script tag as early as possible in the `<head>` of the page where checkout will be rendered, so the SDK can preload resources needed for fast checkout rendering.

        ```html
        <script type="text/javascript" src="https://js.staging.pay.com/v1"></script>
        ```

        <Callout type="info">
          The URL above is for sandbox testing environments only. For production, use `https://js.pay.com/v1` instead.
        </Callout>
      </Tab>
    </Tabs>
  </Step>

  <Step>
  ## Create a payment session
  Before initializing the SDK, your server must create a [payment session](/docs/concepts/payment-sessions). A payment session attaches the order details to a secure checkout and returns two values: a `client_secret`, which authorizes the checkout on the client, and a session `id`, which tells the SDK which payment session to use. Both are required to create a checkout instance.
  
  The example below shows a minimal request. For all available parameters, see the [Payment Sessions API reference](/docs/api-reference/paymentsessions/create-ps).

  <Tabs items={['Universal / Headless', 'Tokenization']}>
    <Tab value="Universal / Headless">
      Call `POST /v1/sessions/payment`. See the full [API reference](/docs/api-reference/paymentsessions/create-ps) for all available parameters.
          ```javascript
            const response = await fetch("https://api.staging.pay.com/v1/sessions/payment", {
              method: "POST",
              headers: {
                "Content-Type": "application/json",
                "x-paycom-api-key": "<YOUR_API_KEY>"
              },
              body: JSON.stringify({
                currency: "usd",
                amount: 2000,
                customer_reference_id: "927071e4-464c-4f5d-a079-4aa097f922ca"
              })
            });

            const session = await response.json();
          ```

      <Callout type="warn">
        `amount` is in **minor units**. Divide by 100 to get the display value. For example, passing `2000` will charge $20.00.
      </Callout>

      A successful response returns a short-lived `client_secret` and session `id`:
        ```json
          {
            "id": "ps_217940156986951680",
            "resource": "payment_session",
            "amount": 1000,
            "currency": "usd",
            "client_secret": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
            "status": "open"
          }
        ```
    </Tab>
    <Tab value="Tokenization">
      Call `POST /v1/sessions/tokenization`to create a tokenization session. See the [Tokenization Session API reference](/docs/api-reference/tokenizationsessions/create-tokenization-session) for the full request and response details.
            ```javascript
            const response = await fetch("https://api.staging.pay.com/v1/sessions/tokenization", {
              method: "POST",
              headers: {
                "Content-Type": "application/json",
                "x-paycom-api-key": "<YOUR_API_KEY>"
              }
            });

            const session = await response.json();
            ```

      A successful response returns a short-lived `client_secret` and session `id`:
      ```json
      {
      "id": "ts_217940156986951680",
      "resource": "tokenization_session",
      "client_secret": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."
      }
      ```
    </Tab>
  </Tabs>
</Step>

  <Step>
    ## Initialize the SDK
    Initialize the SDK using the session `id` returned from your payment session. This returns a `pay` instance, a JavaScript object that represents an active SDK connection you will use to create a checkout instance and render the payment form.

    <Tabs items={['npm / yarn', 'CDN']}>
      <Tab value="npm / yarn">
        Import `Pay` from the `@pay-com/js` package and initialize it with your session `id`.

        ```ts
        import Pay from '@pay-com/js'

        const pay = await Pay.com({
          identifier: '{id}'
        })
        ```
      </Tab>
      <Tab value="CDN">
        When using the CDN, the SDK is available globally on the `window` object. Access `Pay` from `window` and initialize it with your session `id`.

        ```js
        const { Pay } = window

        const pay = Pay.com({
          identifier: '{id}'
        })
        ```
      </Tab>
    </Tabs>

    <Callout type="info">
  By default, the SDK runs in sandbox mode and connects to Pay.com's test environment. Set `live: true` to switch to the production environment, which processes real payments. Omit it during development and testing to avoid charging real cards.
    ```ts
      const pay = await Pay.com({
        identifier: '{id}',
        live: true
      })
    ```
      </Callout>
    
  </Step>
</Steps>
