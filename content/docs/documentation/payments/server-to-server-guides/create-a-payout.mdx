A payout is a backend-only API call that reverses the direction of a standard charge by taking money from the merchant and sending it to the customer.
This flow is commonly used in verticals such as gambling winnings, sports betting, forex withdrawals, and gig-economy platforms paying drivers.

## Requirements

Payouts are not a self-service feature and must be enabled during onboarding. Before sending a payout, ensure the following:

* Your workspace is configured to support payouts by the Pay.com team.
* The destination payment method supports payout flows.
* If you are sending raw card details instead of a stored token, you must be PCI-DSS compliant.

<Steps>
<Step>
## Send the payout request

To send funds, call the [`/v1/payouts`](/docs/api-reference/payouts/create-payout) endpoint from your backend. Amounts are in the smallest currency unit, so `10000` equals $100.00 USD.

You can send a payout to a stored payment method, raw card data, an Apple Pay or Google Pay network token, or a bank account via ACH. Use the `destination` field to pass a stored payment method ID, or `destination_data` to provide raw card or token details.

Pass a stored `pm_us_bank_account_...` ID in the `destination` field. ACH payouts return `status: pending` immediately. The final result arrives asynchronously via webhooks.

```json
{
  "amount": 10000,
  "currency": "usd",
  "destination": "pm_us_bank_account_REPLACE_WITH_BANK_ACCOUNT_PM_ID",
  "reference": "payout-order-005",
  "description": "Payout via ACH bank transfer"
}
```

<Step>
## Handle the response

Unlike charges, payouts may return a `pending` status depending on the destination and network. Check the `status` field in the response:

* `succeeded`: The payout was approved and the funds are being sent.
* `failed`: The payout was declined. No money moved, and `failure_code` and `failure_message` explain the reason.
* `pending`: The request was accepted but is waiting for final bank confirmation. Listen for webhooks for the final result.

</Step>
<Step>
## Listen for webhooks

Because payouts can stay in a `pending` state for several seconds or longer, use webhooks to receive the final result.

* `payout.pending`: The request was accepted and is waiting for network approval.
* `payout.succeeded`: The customer was paid successfully.
* `payout.failed`: The payout was declined. Use this to alert your support team for manual review.

<Callout type="warn">
Always verify webhook signatures using the `Pay-Signature` header and your endpoint's secret key (via HMAC SHA-256) to ensure the event is legitimate.
</Callout>

</Step>
