title: Save a payment method
description: Store a customer's card for future charges using a payment session or setup session. Returns a reusable payment method ID for subscriptions and off-session payments.
---

import { Callout } from 'fumadocs-ui/components/callout'
import { Tab, Tabs } from 'fumadocs-ui/components/tabs'
import { Step, Steps } from 'fumadocs-ui/components/steps'

Saving a payment method gives Pay.com permission to charge a customer's card in the future without the customer needing to be present. Once saved, the card is returned as a reusable payment method ID (`pm_card_xxx`) that you can use for subscriptions and off-session charges.

There are two supported ways to save a payment method:

- **Payment session** — charges the customer and saves their card in a single checkout flow
- **Setup session** — saves the card without an upfront charge, useful for free trials and pay-later flows

In both cases, the customer completes a checkout and their card is returned as a  reusable payment method ID (`pm_card_xxx`).

<Steps>
  <Step>
    ## Create a session on your server

    The first step is to create a session on your server. The session authorizes the checkout on the client side and tells Pay.com to store the card for future use.

    Set `setup_future_usage: off_session` in your session request. This tells Pay.com the card will be charged in the future without the customer present.

    <Callout type="info">
      `off_session` is required for subscriptions and any recurring charge where the customer will not be present at checkout. If you omit it, the stored card will not be eligible for future off-session charges.
    </Callout>

    <Tabs items={['Payment session', 'Setup session']}>
      <Tab value="Payment session">
        A payment session charges the customer and saves their card in a single checkout flow. Use this when you want to charge the customer upfront and store their card for future use at the same time.

        Send a `POST` request to `/v1/sessions/payment` with `setup_future_usage: off_session`:

        ```javascript
        const response = await fetch("https://api.staging.pay.com/v1/sessions/payment", {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            "x-paycom-api-key": "<YOUR_API_KEY>"
          },
          body: JSON.stringify({
            amount: 1000,
            currency: "usd",
            customer_reference_id: "927071e4-464c-4f5d-a079-4aa097f922ca",
            setup_future_usage: "off_session"
          })
        })
        ```

        A successful response returns a `client_secret` and session `id`. Pass both to the SDK in the next step.

        ```json
        {
          "id": "ps_296645958265997319",
          "resource": "payment_session",
          "client_secret": "eyJ0eXAiOiJKV1Qi...",
          "customer_reference_id": "927071e4-464c-4f5d-a079-4aa097f922ca",
          "status": "open"
        }
        ```
      </Tab>

      <Tab value="Setup session">
        A setup session saves the customer's card without charging them. Use this for free trials, pay-later flows, or any scenario where you want to collect card details upfront without an immediate charge.

        Send a `POST` request to `/v1/sessions/setup` with `setup_future_usage: off_session`:

        ```javascript
        const response = await fetch("https://api.staging.pay.com/v1/sessions/setup", {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            "x-paycom-api-key": "<YOUR_API_KEY>"
          },
          body: JSON.stringify({
            customer_reference_id: "927071e4-464c-4f5d-a079-4aa097f922ca",
            setup_future_usage: "off_session"
          })
        })
        ```

        A successful response returns a `client_secret` and session `id`. Pass both to the SDK in the next step.

        ```json
        {
          "id": "ss_296645958265997319",
          "resource": "setup_session",
          "client_secret": "eyJ0eXAiOiJKV1Qi...",
          "customer_reference_id": "927071e4-464c-4f5d-a079-4aa097f922ca",
          "status": "open"
        }
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step>
    ## Complete the checkout

    After creating the session, initialize the Pay.com SDK on your frontend using the session `id` and `client_secret` returned from your server.
    The SDK renders a card form for the customer to enter their checkout details.

    The flow is the same for both session types, the SDK initializes identically whether you used a payment session or a setup session:

    ```javascript
    const pay = await Pay.com({
      identifier: '{id}',
    })

    const checkout = pay.checkout({
      clientSecret: '{client_secret}',
      onSuccess: (payment) => {
        // payment.amount        — 0 for setup sessions, charged amount for payment sessions
        // payment.customer      — unique Pay.com customer ID
        // payment.paymentMethod — stored payment method ID (pm_card_xxx)
        // payment.status        — "succeeded"
        // payment.transaction   — charge ID (chrg_xxx) or setup session ID (ss_xxx)
        console.log('Payment method saved:', payment.paymentMethod)
      },
      onFailure: (error) => {
        console.log('Failed:', error.message)
      },
    })

    checkout.universal({ container: '#paycom_checkout' })
    ```

    <Callout type="info">
      For a setup session, the customer enters their card details and submits. No charge is made.
    </Callout>
  </Step>

  <Step>
    ## Save the payment method ID

    Once the customer completes the checkout, `payment.paymentMethod` in the `onSuccess` callback returns the stored payment method ID (`pm_card_xxx`). Save this value in your database against the customer, it is what you will use to charge the customer in future billing cycles.

    ```javascript
    onSuccess: (payment) => {
      const paymentMethodId = payment.paymentMethod //(pm_card_684790996696828928)
      // Save paymentMethodId to your database against the customer
    }
    ```

    <Callout type="info">
      Pay.com stores card payment methods as network tokens. If a customer's card is reissued with a new number or expiry date, the token updates automatically, meaning future charges continue without the customer needing to update their details.
    </Callout>
  </Step>
</Steps>

Once you have the payment method ID, you can use it to set up a subscription or charge the customer directly without them needing to be present.
