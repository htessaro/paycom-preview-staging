This guide covers how to trigger wallet and bank redirect payment methods using the Headless SDK. Each payment method is triggered by calling `checkout.headless()` with the payment method type after the `checkout` instance has been created.

<Callout type="info">
   If you are using the Universal SDK, all activated payment methods are rendered automatically. No additional code is needed beyond what is covered in [create a charge with the SDK](/docs/payments/sdks/create-a-charge-with-sdk) guide.
</Callout>

## Requirements

Before you begin, make sure you have:

- A `client_secret` from your payment session
- An initialized `pay` instance

If you haven't completed these steps yet, see the [get started with the Pay.com SDK](/docs/payments/sdks/integrate-the-pay-com-sdk) guide to create a payment session and initialize the SDK.


## Payment methods

Apple Pay, Google Pay, iDEAL, and GiroPay must be activated on your Pay.com merchant account before use. Check the Payment methods section of your [dashboard](https://dashboard.pay.com) to confirm which methods are available on your account.

### Apple Pay and Google Pay

Before rendering wallet buttons in your UI, check whether Apple Pay or Google Pay are available in the customer's browser using `canMakePayments()`. It returns a boolean per wallet so you can conditionally show the relevant button.
```js
const canMakePaymentsResult = await checkout.canMakePayments()
// { applepay: true, googlepay: true }
```

Once you have confirmed availability, trigger the relevant payment when the customer clicks the button:
```js
async function handleApplePay() {
  await checkout.headless('applepay')
}

async function handleGooglePay() {
  await checkout.headless('googlepay')
}
```

### iDEAL

iDEAL is a bank redirect method that requires the customer's bank BIC code.
```js
async function handleIDEAL() {
  await checkout.headless('ideal', {
    bic: 'SNSBNL2A',
  })
}
```

### GiroPay

GiroPay is a bank redirect method that requires no additional data.
```js
async function handleGiroPay() {
  await checkout.headless('giropay')
}
```

## Accept Apple Pay and Google Pay

The example below demonstrates how Apple Pay and Google Pay appear in a React application using the Headless SDK checkout. If you want to include card payments alongside wallet methods, see the full component example in [create a charge with the SDK](/docs/payments/sdks/create-a-charge-with-sdk) guide.

```jsx
import { useEffect, useState, useRef } from 'react'
import Pay from '@pay-com/js'

function WalletCheckout() {
  const checkoutRef = useRef(null)
  const [wallets, setWallets] = useState({ applepay: false, googlepay: false })
  const [paymentDone, setPaymentDone] = useState(false)
  const [errorMessage, setErrorMessage] = useState(null)

  useEffect(() => {
    async function initCheckout() {
      const pay = await Pay.com({
        identifier: '{id}',
      })

      const checkout = pay.checkout({
        clientSecret: '{client_secret}',
        onSuccess: (payment) => {
          setPaymentDone(true)
        },
        onFailure: (error) => {
          setErrorMessage(error.message)
        },
      })

      checkoutRef.current = checkout

      // Check wallet availability after checkout is created
      const canMakePayments = await checkout.canMakePayments()
      setWallets(canMakePayments)
    }

    initCheckout()
  }, [])

  async function handleApplePay() {
    await checkoutRef.current.headless('applepay')
  }

  async function handleGooglePay() {
    await checkoutRef.current.headless('googlepay')
  }

  if (paymentDone) {
    return <p>Payment complete. Thank you for your order.</p>
  }

  return (
    <div>
      {wallets.applepay && (
        <button onClick={handleApplePay}>Apple Pay</button>
      )}
      {wallets.googlepay && (
        <button onClick={handleGooglePay}>Google Pay</button>
      )}
      {!wallets.applepay && !wallets.googlepay && (
        <p>No wallet payment methods available in this browser.</p>
      )}
      {errorMessage && <p style={{ color: 'red' }}>{errorMessage}</p>}
    </div>
  )
}

export default WalletCheckout
```

## Webhooks

Wallet and bank redirect payments use the same webhook event as card payments. See [handling payments and webhooks](/docs/payments/sdks/create-a-charge-with-sdk#webhooks) for details.
