Use these card numbers to simulate payment scenarios in the
[sandbox environment](/api-reference/get-started/environments). Test cards
behave exactly like real cards in the API, but no funds are moved.

<Callout type="warn">
Test cards only work in the sandbox environment. Using them in production
will result in a declined transaction.
</Callout>

When using any test card, you can supply:

- **Expiry date:** any future date (for example, `12/2028`)
- **CVV:** any 3-digit number (for example, `123`), or 4 digits for
  American Express cards
- **Billing address:** any valid address

## Approved cards

These cards simulate a successful authorization. Use them to test the
happy path: a charge that completes without friction or additional
authentication steps.

## Declined cards

These cards simulate a declined transaction. Use them to verify that your
integration handles payment failures correctly, for example, showing the
right error message to the customer or prompting them to try a different
card.

| Card number | Decline reason |
| :--- | :--- |
| `4000007818415900` | Do not honor |
| `4000005357086322` | Insufficient funds |
| `4000004452879855` | Stolen card |

## 3DS result cards

These cards give you predictable 3D Secure outcomes without going through
a full interactive 3DS challenge. Use them to verify that your integration
responds correctly to each result, for example, redirecting the customer
when a challenge is needed or handling authentication failures gracefully.

| Card number | Result |
| :--- | :--- |
| `4000000000001000` | Approved |
| `5200000000001096` | 3DS challenge, approved |
| `4000000000001091` | 3DS challenge, declined (insufficient funds) |
| `4000000000001109` | 3DS challenge failure |
