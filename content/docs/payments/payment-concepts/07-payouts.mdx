---
title: Payouts
description: Distribute funds from the merchant account to customer banks or wallets
---

A Payout reverses the direction of a standard charge by taking funds from your merchant account and sending them directly to a customer's bank account or digital wallet.

## Creating a payout

To initiate a payout, send a `POST` request to the [`/v1/payouts`](/docs/api-reference/payouts/create-payout) endpoint.

When creating a payout, you must specify the destination using one of these methods:

* **Stored payment method:** Use a previously vaulted payment method ID (such as a stored card or bank account)
* **Raw destination details:** Pass destination information using the `destination_data` object

The API supports various destinations including raw card details, Apple Pay, Google Pay, and PayPal. Each payment method includes an `enable_flows` parameter that indicates whether it's eligible for payout transactions.

## Asynchronous processing

Unlike immediate card authorizations, some payout methods require background processing and don't provide instant results.

Bank transfers and similar methods may return a `pending` status instead of immediate success or failure. You must configure webhooks to receive notifications when the banking network updates the system with the final result (typically within seconds or minutes).

## Retrieving payouts

Query payout data using these methods:

* **Specific payout:** Use the payout's unique identifier to retrieve its details and status
* **All payouts:** Use `GET /v1/payouts` to retrieve a list for reporting and reconciliation
* **Dashboard:** Initiate and track payouts manually through the Pay.com dashboard

## Webhook events

Configure webhooks to receive the final result of asynchronous payouts:

* `payout.pending`: Awaiting final bank confirmation
* `payout.succeeded`: Funds successfully sent to the customer
* `payout.failed`: Payout request failed; no funds were moved
